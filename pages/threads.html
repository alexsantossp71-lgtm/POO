Threads e Concorrência

Fazendo várias coisas ao mesmo tempo

Multitarefa em Java

Processadores modernos possuem múltiplos núcleos (cores). Se seu programa usa apenas uma linha de execução (Single Thread), ele está desperdiçando poder de processamento. Threads permitem que partes do seu código rodem em paralelo.

Criando Threads

A maneira preferida é implementar a interface Runnable e passá-la para um objeto Thread. 


// Define a tarefa
class MinhaTarefa implements Runnable {
    @Override
    public void run() {
        for(int i=0; i<5; i++) {
            System.out.println("Tarefa rodando: " + i);
            try { Thread.sleep(1000); } catch (InterruptedException e) {}
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Thread t1 = new Thread(new MinhaTarefa());
        t1.start(); // Inicia em paralelo!
        
        System.out.println("Main continua livre...");
    }
}

O Perigo da Concorrência

Quando duas threads tentam alterar a mesma variável ao mesmo tempo, pode ocorrer uma Race Condition (Condição de Corrida), gerando dados corrompidos. Para evitar isso, usamos a palavra-chave synchronized. 


class Conta {
    private double saldo = 100;

    // Apenas uma thread entra aqui por vez!
    public synchronized void sacar(double valor) {
        if(saldo >= valor) {
            saldo -= valor;
        }
    }
}

Problemas Comuns

Deadlock: Acontece quando duas threads ficam esperando uma pela outra eternamente. Ninguém avança.

Race Condition: Quando o resultado depende da sorte (quem chega primeiro). O synchronized resolve isso.

ExecutorService (Pool de Threads)

Criar threads é custoso para o sistema. Em vez de criar uma thread para cada tarefa (new Thread()), usamos um Pool de Threads que reaproveita threads existentes. 


ExecutorService executor = Executors.newFixedThreadPool(10); // 10 threads fixas
executor.submit(new MinhaTarefa()); // Envia tarefa para a fila
executor.shutdown(); // Encerra quando acabar

Estados da Thread

New

Criada, mas não iniciada (antes do start).

Runnable

Executando ou pronta para executar.

Blocked/Waiting

Esperando recurso ou tempo (sleep).
