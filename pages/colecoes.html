O Java Collections Framework

Em aplicações reais, raramente trabalhamos com um único objeto. Precisamos de listas de usuários, filas de tarefas, conjuntos de produtos únicos, etc. O Java fornece um conjunto poderoso de interfaces e classes para lidar com isso de forma eficiente.

List (Listas)

Coleções ordenadas que permitem elementos duplicados. O usuário tem controle total sobre onde cada elemento é inserido. 


// ArrayList: Rápido para leitura, lento para inserção no meio
List herois = new ArrayList<>();
herois.add("Batman");
herois.add("Superman");
herois.add("Batman"); // Aceita duplicatas

System.out.println(herois.get(0)); // Acesso por índice: Batman

Set (Conjuntos)

Coleções que não permitem duplicatas. A ordem dos elementos geralmente não é garantida (HashSet). 


// HashSet: Muito rápido, sem ordem garantida
Set numeros = new HashSet<>();
numeros.add(10);
numeros.add(20);
numeros.add(10); // Ignorado!

System.out.println(numeros.size()); // 2

Map (Mapas)

Não herda de Collection. Mapeia Chaves para Valores. Chaves não podem ser duplicadas. 


// HashMap: Chave -> Valor
Map capitais = new HashMap<>();
capitais.put("SP", "São Paulo");
capitais.put("RJ", "Rio de Janeiro");

System.out.println(capitais.get("SP")); // São Paulo

Qual usar?

ArrayList

Use quando precisar acessar elementos por posição (índice).

HashSet

Use quando precisar garantir unicidade (sem repetidos).

HashMap

Use para dicionários de dados (buscar valor por chave).
