Abstração

Simplificando a complexidade.

O que é?

Abstração é o processo de focar apenas nas características essenciais de um objeto, ignorando detalhes de implementação menos importantes para o contexto atual. Em Java, a abstração é alcançada principalmente através de Classes Abstratas e Interfaces.

Uma classe abstrata não pode ser instanciada diretamente; ela serve como um modelo base para outras classes.

Exemplo em Java

// Classe Abstrata: Define um conceito genérico
abstract class FormaGeometrica {
    String cor;
    
    // Método abstrato: As subclasses SÃO OBRIGADAS a implementar
    abstract double calcularArea();
    
    // Método concreto: Já tem implementação e pode ser herdado
    void exibirCor() {
        System.out.println("Cor: " + cor);
    }
}

// Classe Concreta
class Circulo extends FormaGeometrica {
    double raio;
    
    Circulo(double raio, String cor) {
        this.raio = raio;
        this.cor = cor;
    }
    
    @Override
    double calcularArea() {
        return Math.PI * raio * raio;
    }
}

// Classe Concreta
class Retangulo extends FormaGeometrica {
    double largura, altura;
    
    Retangulo(double l, double a, String cor) {
        this.largura = l;
        this.altura = a;
        this.cor = cor;
    }
    
    @Override
    double calcularArea() {
        return largura * altura;
    }
}

Pontos Chave

  • abstract: Palavra-chave para definir classes e métodos abstratos.
  • Interface: Um contrato que define um conjunto de métodos que uma classe deve implementar. É um nível mais alto de abstração (100% abstrata até o Java 7).
  • Foco no "O Que": A abstração define O QUE um objeto faz, não COMO ele faz.
