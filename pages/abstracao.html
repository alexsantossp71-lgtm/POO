O Poder da Simplificação

Quando você dirige um carro, você não precisa saber como o motor de combustão interna funciona, como a injeção eletrônica calcula a mistura de ar/combustível ou como o diferencial distribui o torque. Você só precisa saber usar o volante, os pedais e o câmbio.

Isso é Abstração: esconder os detalhes complexos de implementação e expor apenas uma interface simples e essencial para o usuário.

Classes Abstratas

Em Java, usamos a palavra-chave abstract para criar classes que não podem ser instanciadas diretamente. Elas servem como modelos incompletos para outras classes. 


// Classe Abstrata: Define o conceito genérico de "Forma"
abstract class Forma {
    String cor;
    
    // Método abstrato: Não tem corpo!
    // As subclasses SÃO OBRIGADAS a implementar isso.
    abstract double calcularArea();
    
    // Método concreto: Já funciona e é herdado por todos
    void mostrarCor() {
        System.out.println("A cor é " + cor);
    }
}

class Circulo extends Forma {
    double raio;
    
    // Implementação obrigatória
    @Override
    double calcularArea() {
        return Math.PI * raio * raio;
    }
}

Interfaces

Uma Interface é como um contrato. Ela define o que deve ser feito, mas não como. Uma classe pode implementar múltiplas interfaces (diferente de herança, onde só pode estender uma classe). 


interface Pagavel {
    void processarPagamento(double valor);
}

class CartaoCredito implements Pagavel {
    @Override
    public void processarPagamento(double valor) {
        System.out.println("Pagando R$" + valor + " no crédito.");
    }
}

class Pix implements Pagavel {
    @Override
    public void processarPagamento(double valor) {
        System.out.println("Transferindo R$" + valor + " via Pix.");
    }
}
