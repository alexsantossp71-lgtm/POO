Sistemas Distribuídos com Sockets

O Mundo Conectado

Aplicações modernas raramente funcionam isoladas. Elas precisam conversar com bancos de dados, APIs externas ou outros clientes. O Java fornece o pacote java.net para facilitar essa comunicação via protocolo TCP/IP.

Sockets: O Canal de Comunicação

Um Socket é como um plugue de telefone. Para haver uma ligação, precisamos de dois sockets conectados: um no cliente e outro no servidor. 


// --- SERVIDOR (Ouve na porta 5000) ---
ServerSocket server = new ServerSocket(5000);
System.out.println("Aguardando conexão...");

Socket socket = server.accept(); // Bloqueia até alguém conectar
System.out.println("Cliente conectou!");

// --- CLIENTE (Conecta no localhost:5000) ---
Socket socket = new Socket("127.0.0.1", 5000);

Trocando Mensagens

Uma vez conectados, usamos Streams (Input e Output) para enviar e receber dados, exatamente como fazemos com arquivos. 


// Enviando dados (Lado A)
PrintStream saida = new PrintStream(socket.getOutputStream());
saida.println("Olá do Java!");

// Recebendo dados (Lado B)
Scanner entrada = new Scanner(socket.getInputStream());
String mensagem = entrada.nextLine();
System.out.println("Recebi: " + mensagem);

Protocolos

TCP

Confiável, orientado a conexão. Garante que os dados cheguem na ordem certa (ex: Web, Email).

UDP

Rápido, sem conexão. Pode perder pacotes. Usado em jogos e streaming de vídeo.
