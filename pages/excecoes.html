Exceções

Gerenciando erros e imprevistos.

O que são?

Exceções são eventos anormais que ocorrem durante a execução de um programa e interrompem o fluxo normal das instruções. O Java fornece um mecanismo robusto para capturar e tratar esses erros, evitando que o programa feche abruptamente.

Exemplo em Java

public class ExemploExcecao {
    public static void main(String[] args) {
        try {
            // Código que pode gerar erro
            int[] numeros = {1, 2, 3};
            System.out.println(numeros[10]); // Índice inválido!
            
        } catch (ArrayIndexOutOfBoundsException e) {
            // Código executado se ocorrer o erro específico
            System.out.println("Erro: Você tentou acessar uma posição que não existe.");
            
        } catch (Exception e) {
            // Código para qualquer outro erro genérico
            System.out.println("Ocorreu um erro inesperado: " + e.getMessage());
            
        } finally {
            // Código que SEMPRE executa, com erro ou não
            System.out.println("Fim da verificação.");
        }
    }
}

Conceitos Importantes

  • try: Bloco de código onde o erro pode ocorrer.
  • catch: Bloco que captura e trata o erro.
  • finally: Bloco opcional que sempre executa (útil para fechar arquivos ou conexões).
  • throw: Usado para lançar uma exceção manualmente.
  • throws: Indica que um método pode lançar uma exceção e quem chamar deve tratar.
