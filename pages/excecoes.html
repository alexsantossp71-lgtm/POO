Tratamento de Exceções

Lidando com o inesperado de forma elegante

O Que Pode Dar Errado?

Arquivos que não existem, internet que cai, divisão por zero, memória cheia. Erros acontecem. Um bom software não é aquele que nunca falha, mas aquele que sabe se recuperar quando algo dá errado.

Em Java, usamos Exceções para sinalizar problemas. Se não tratadas, elas derrubam o programa. Se tratadas corretamente, o programa continua rodando ou avisa o usuário amigavelmente.

Try, Catch e Finally

A estrutura básica de proteção é envolver o código perigoso em um bloco try. 


public class LeitorArquivo {
    public void lerDados() {
        try {
            // Código arriscado
            int[] numeros = {1, 2, 3};
            System.out.println(numeros[10]); // Erro! Índice não existe.
            
        } catch (ArrayIndexOutOfBoundsException e) {
            // O que fazer se der erro?
            System.out.println("Ops! Você tentou acessar uma posição inválida.");
            
        } catch (Exception e) {
            // Captura genérica para outros erros
            System.out.println("Erro desconhecido: " + e.getMessage());
            
        } finally {
            // Executa SEMPRE (com sucesso ou erro)
            System.out.println("Finalizando operação...");
        }
    }
}

Checked vs Unchecked

  • Unchecked Exceptions (RuntimeException)

    Erros de lógica do programador (ex: NullPointerException). O compilador não obriga a tratar.

  • Checked Exceptions

    Erros externos previsíveis (ex: IOException, SQLException). O compilador obriga a tratar com try-catch ou throws.

Hierarquia e Palavras-Chave

Tudo herda de Throwable.

  • Error: Problemas graves da JVM (ex: OutOfMemoryError). Não devemos tentar tratar.
  • Exception: Problemas que podem ser tratados pela aplicação.

Throw vs Throws

throw: É um comando (verbo). Usado para lançar uma exceção agora.
throw new IllegalArgumentException("Idade inválida");

throws: É um aviso (declaração). Usado na assinatura do método para avisar que ele pode lançar um erro.
public void lerArquivo() throws IOException { ... }
