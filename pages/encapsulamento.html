Encapsulamento

Protegendo e controlando o acesso aos dados.

O que é?

Encapsulamento é um dos pilares da POO que consiste em ocultar os detalhes internos de uma classe e expor apenas o necessário. Isso é feito restringindo o acesso direto aos atributos da classe e fornecendo métodos públicos (Getters e Setters) para manipulá-los de forma segura.

Isso protege a integridade dos dados, impedindo que o objeto entre em um estado inválido (por exemplo, uma idade negativa).

Exemplo em Java

public class ContaBancaria {
    // Atributo privado: Ninguém de fora pode acessar diretamente
    private double saldo;
    private String titular;

    public ContaBancaria(String titular) {
        this.titular = titular;
        this.saldo = 0.0;
    }

    // Método público para depositar (com validação)
    public void depositar(double valor) {
        if (valor > 0) {
            this.saldo += valor;
            System.out.println("Depósito de R$" + valor + " realizado.");
        } else {
            System.out.println("Valor inválido para depósito.");
        }
    }

    // Método público para sacar (com validação)
    public void sacar(double valor) {
        if (valor > 0 && valor <= this.saldo) {
            this.saldo -= valor;
            System.out.println("Saque de R$" + valor + " realizado.");
        } else {
            System.out.println("Saldo insuficiente ou valor inválido.");
        }
    }

    // Getter: Permite apenas LER o saldo, não alterar
    public double getSaldo() {
        return saldo;
    }
}

Modificadores de Acesso

  • private: Acessível apenas dentro da própria classe.
  • public: Acessível de qualquer lugar.
  • protected: Acessível no mesmo pacote e por subclasses.
  • default (sem modificador): Acessível apenas no mesmo pacote.
