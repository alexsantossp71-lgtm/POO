Protegendo o Estado do Objeto

Imagine uma cápsula de remédio. Você não vê o pó químico lá dentro, nem toca nele diretamente. Você interage apenas com a cápsula. Isso é Encapsulamento.

Na programação, encapsular significa esconder os detalhes internos de como uma classe funciona e expor apenas o que é seguro e necessário. Fazemos isso declarando atributos como private e criando métodos públicos (Getters e Setters) para acessá-los.

Por que usar?

Se deixarmos os atributos públicos, qualquer um pode colocar valores inválidos (ex: idade negativa, saldo bancário infinito). Com encapsulamento, podemos validar os dados antes de aceitá-los.

Exemplo: Conta Bancária


public class ContaBancaria {
    // Atributos privados (Ocultos)
    private double saldo;
    private String titular;

    public ContaBancaria(String titular) {
        this.titular = titular;
        this.saldo = 0.0;
    }

    // Método público controlado para depositar
    public void depositar(double valor) {
        if (valor > 0) {
            this.saldo += valor;
            System.out.println("Sucesso! Novo saldo: " + this.saldo);
        } else {
            System.out.println("Erro: Valor deve ser positivo.");
        }
    }

    // Getter: Permite VER o saldo, mas não alterar diretamente
    public double getSaldo() {
        return saldo;
    }
}

Modificadores de Acesso

  • private Visível apenas dentro da própria classe.
  • public Visível para todo o projeto.
  • protected Visível para subclasses e mesmo pacote.

Benefícios

Segurança

Impede estados inconsistentes.

Flexibilidade

Podemos mudar a implementação interna sem quebrar quem usa a classe.
