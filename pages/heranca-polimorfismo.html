Herança e Polimorfismo

Estendendo comportamentos e flexibilizando o código

Herança (Inheritance)

Herança permite que uma classe (Subclasse) herde atributos e métodos de outra classe (Superclasse). Isso promove o reaproveitamento de código. Em Java, usamos a palavra-chave extends.

Importante: Java não suporta herança múltipla de classes (uma classe só pode ter um pai). 


class Animal {
    void comer() {
        System.out.println("Comendo...");
    }
}

// Cachorro herda de Animal
class Cachorro extends Animal {
    void latir() {
        System.out.println("Au Au!");
    }
}

Polimorfismo

Significa "muitas formas". É a capacidade de tratar objetos de subclasses diferentes como se fossem da superclasse genérica.

1. Sobrescrita (Overriding)

Quando a subclasse redefine um método da superclasse. Usamos a anotação @Override. A assinatura do método deve ser idêntica.

2. Sobrecarga (Overloading)

Ter vários métodos com o mesmo nome na mesma classe, mas com parâmetros diferentes. 


class Calculadora {
    // Sobrecarga: Somar inteiros
    int somar(int a, int b) { return a + b; }
    
    // Sobrecarga: Somar decimais
    double somar(double a, double b) { return a + b; }
}

Conceitos Chave

extends

Palavra-chave para aplicar herança.

super

Referência à superclasse (pai). Usado para chamar construtores ou métodos do pai.

Upcasting

Tratar uma subclasse como superclasse (ex: Animal a = new Cachorro();).
