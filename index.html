O que é POO?

A Programação Orientada a Objetos é um paradigma de programação que organiza o código em objetos que representam entidades do mundo real. Cada objeto possui características (atributos) e comportamentos (métodos), permitindo criar sistemas mais modulares, reutilizáveis e fáceis de manter.

Conceitos Principais

Classes e Objetos

Classes são moldes que definem a estrutura e comportamento dos objetos. Objetos são instâncias de classes com características únicas.

Fundamento

Encapsulamento

Oculta os detalhes internos de implementação, expondo apenas interfaces necessárias através de modificadores de acesso (private, protected, public).

Proteção

Abstração

Simplifica a complexidade ao representar apenas os aspectos essenciais, ignorando detalhes irrelevantes para o contexto.

Simplicidade

Exceções

Gerenciamento robusto de erros em tempo de execução, permitindo que programas lidem graciosamente com situações inesperadas.

Confiabilidade

Coleções

Estruturas de dados como listas, filas, pilhas e árvores para organizar e manipular grupos de objetos de forma eficiente.

Estruturas

Streams & Serialização

Comunicação e persistência de dados através de streams, incluindo serialização e desserialização de objetos.

I/O

Threads

Programação concorrente permitindo que múltiplas tarefas sejam executadas simultaneamente para melhor performance.

Concorrência

Redes

Desenvolvimento de aplicações distribuídas usando sockets, protocolos TCP/UDP e arquitetura cliente-servidor.
